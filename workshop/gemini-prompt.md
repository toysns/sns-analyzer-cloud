## スライド10: 伸びる構造②
タイトル: 定数と変数
本文:
定数（変えてはダメ）:
ジャンル / 市場 / 発信者のキャラ / ターゲット / プラットフォーム特性

変数（テストして改善）:
フック / 構成 / 演出 / CTA / 見せ方 / 頻度

判定法: 「これを変えたら、視聴者が本当に求めているもの（需要A）に応えられなくなるか？」
→ Yes なら定数。No なら変数。

成功パターン: 変数のみ変更し、定数は維持 → 差別化しつつ需要に応える
失敗パターン: 定数を壊している → 架空の需要にしかヒットしない

## スライド13: コンセプト設計①
タイトル: コンセプトとは何か
本文:
コンセプト ≠ 世界観っぽい抽象語

コンセプト = "誰の、どんな感情・課題に対して、どんな価値を、どの立ち位置で届けるか"

つまり:
- Who: 誰が発信しているか
- What: 何を届けたいか
- How: どうやって届けるか

この3つが明確であること。

## スライド14: コンセプト設計②
タイトル: コンセプトがあると / ないと
本文（左右対比）:

【コンセプトがあると】
- 何を発信するかが決まる
- 誰に刺さるかが定まる
- フォーマットの選び方がブレない
- 運用の一貫性が出る
- 施策の良し悪しを定数/変数で判断できる

【コンセプトがないと】
- 毎回ネタ探しになる
- 数字の良し悪しを解釈できない
- たまたま当たっても再現しない
- バズっても応援されない

事例: 心理トリガーは上手く使えるが思想がないアカウント → 再生は取れるがフォロワーが増えない。一時的なバズで終わる。

## スライド19: 思考フロー③

【供給の分解】

## スライド20: 思考フロー④
タイトル: Step 6〜7: フォーマット選定と検証
本文:
【リズム同調の法則】
自分が日常的に見ているフォーマットを選ぶ。
→ 見てないフォーマットはリズム感覚が身についておらず、品質判断ができない。
→ Vlogを見ない人がVlogをやっても伸びない。

【内容と構成を分けて評価する】

| 評価対象 | 指標 | 判断基準 |
|---|---|---|
| 内容（ネタ） | 初速エンゲージメント率 | 反応が悪ければネタが悪い |
| 構成（見せ方） | 冒頭3秒の維持率 | 維持率50%以上で合格、30%未満なら即改善 |

内容が良い × 構成が良い → バズる
内容が良い × 構成が悪い → 構成を変えて再投稿の価値あり
内容が悪い × 構成が良い → ネタを変える
内容が悪い × 構成が悪い → 全体やり直し

## スライド22: ケース分解②
タイトル: ケース分解 — 分解と改善
本文:
【伸びている理由の分解】
- 定数を維持しながら、変数で差別化できているか
- どの心理トリガーが機能しているか

【伸びていない場合の診断】
- 定数を壊していないか
- 思想（Who × What × How）は明確か

【改善の優先度】
1. 致命的: 思想不在 / 定数破壊 / 視聴者の質 / マネタイズ導線なし
2. 重要: フォロワー転換率 / エンゲージメント / プロフィール / イントロ
3. 改良余地: 編集 / 投稿頻度 / コンテンツバリエーション

## スライド23: まとめ
タイトル: 明日から持ち帰る4つの視点
本文:
1. 投稿を見る前に、まずコンセプトを見る
   → Who × What × How は明確か

2. 当たり外れではなく、定数/変数で振り返る
   → 内容と構成を分けて評価する

3. バズではなく、再現可能な設計を目指す
   → 伸びた型は擦り続ける。感覚ではなくデータで判断

4. その先にいるのは「人」
   → クライアントの顔色ではなく、届ける相手と直接話す
   → 初めてアカウントが伸びた時の純粋な気持ちを忘れない
