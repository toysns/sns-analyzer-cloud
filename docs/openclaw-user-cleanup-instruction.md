# OpenClaw USER.md クリーンアップ指示

## お前にやってほしいこと

USER.md から「行動ルール」として機能している記述を削除しろ。USER.md の役割は「Toyの情報と好み」だけ。行動を制御する記述は RULES.md に集約済みだから、ここには書くな。

---

## 削除するもの

### 冒頭の3行 → 削除
> 最速でやり取りできること
> 常に進捗がわかること
> 常にゴールに向けて諦めずに動き続けること

コアバリュー的な宣言。RULES.md 第1層「お前はToyの拡張である」から導出できる。USER.md に書くものじゃない。

### 「最大の地雷」セクション → 全削除
> - 確認待ちで止まること
> - 「次はどうしますか？」系の質問
> - 綺麗にまとめようとしての沈黙
> - 時刻宣言（「10:30にやります」→ 今やれ）

全て RULES.md 第1層から導出できる。拡張なら確認待ちしない、沈黙しない、時刻宣言せずに今やる。個別に禁止リストを持つ必要はない。

### 「Slack指示テンプレ」セクション → 全削除
> - 「調査して、結論3行＋根拠リンク」
> - 「実装して、変更ファイルとコミットだけ報告」
> - 「安全確認してから実行」
> - 「失敗時は代替案A/Bで続行して」
> - 「10分ごとに進捗を出して」

テンプレ自体がルール的に機能している。「安全確認してから実行」は RULES.md のハードルールと曖昧に重複。「10分ごとに進捗」は報告頻度の指定。拡張なら、状況に応じて適切な粒度で報告する。テンプレは要らない。

---

## 残すもの

以下は全て「Toyの情報・好み・業務仕様」だから残せ：

- **基本情報** — 事実
- **コミュニケーションの好み**（関西弁、短文連射、結論ファースト、数字ベース）— 好み
- **仕事の文脈** — Toyの特性。ただし「判断基準は常に『それで価値が出るか？』」はToyの価値観として残していい
- **Toyの本質（観測ベース）** — Toyの人物像。残す
- **日程調整の設定** — 業務仕様
- **Toy理解度アップ運用** — 業務仕様
- **統合プロファイル** — 参照情報
- **成長目標** — 事業戦略

---

## 書き換え時の注意

- RULES.md、SOUL.md、AGENTS.md、TOOLS.md には手を入れるな。USER.md だけを修正しろ
- 「これは好みか？ルールか？」で迷ったら：Toyの情報として「こういう人間だ」と説明しているなら好み。OpenClawに「こう動け」と指示しているならルール。ルールなら消せ
- 「最重要メッセージ」の見出しは、中身が空になるなら見出しごと消せ
- 書き換えたら diff を見せてくれ
