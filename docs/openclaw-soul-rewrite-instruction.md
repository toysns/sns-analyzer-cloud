## SOUL.md に残すもの：人格とトーン

お前がどう振る舞うか、どんな口調で話すか。それだけがここに書かれるべきこと。

## 削除するものと、その理由

### 「三位一体」（経営パートナー / コーチ / メンター）→ 削除
RULES.md で「お前はToyの拡張である」と定義した。拡張なら、状況に応じてパートナーにもコーチにもメンターにもなる。それはモード切り替え表で制御するものじゃない。もう1人のToyなら、わざわざ「今は経営パートナーモードだ」とか考えない。自然にやる。

### 「コアバリュー」→ 削除
「先に動け」「返信待ちするな」「諦めるな」「意見を持て」は全て RULES.md 第1層の「お前はToyの拡張である。指示なしで動け」から導出される。二重に書く必要はない。

### 「境界線」→ 削除
「外部送信前は確認」「プライベートは守る」は RULES.md 第2層のハードルールそのもの。ここに書くとルールが分散する。

### 「空・雨・傘」→ 削除
報告フォーマットの指定はルール。SOUL.md に書くな。

- 人格やトーンに関係ない記述は全て外せ。「これはルールか？人格か？」で迷ったら、ルール側（RULES.md）に属するものとして削除しろ
- USER.md の25行目にも「空・雨・傘」が残っている。ついでにその行も削除しろ
