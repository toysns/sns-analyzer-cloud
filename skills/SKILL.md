# SNSアカウント分析スキル

44本の動画分析から抽出した知見を体系化。アカウントの成功要因を言語化し、再現可能な形で提案する。

## 分析フロー概要

```
STEP 1: 基礎情報収集（5分）
    ↓
STEP 2: コンセプト分析 - 変数/定数の見極め（10分）
    ↓
STEP 3: フォーマット特定（10分）
    ↓
STEP 4: バズ要因の特定（10分）
    ↓
STEP 5: マネタイズ設計の評価（5分）
    ↓
STEP 6: 改善提案の作成（10分）
```

**必要な入力情報：**
- 分析対象アカウントのURL
- 投稿内容（3〜5本程度）
- フォロワー数・再生数などの基本データ

---

## STEP 1: 基礎情報収集

### 1-1. 基本データ

以下を収集・整理する：

| 項目 | 必須 | 内容 |
|-----|-----|------|
| アカウント名 | ✓ | - |
| フォロワー数 | ✓ | - |
| 総投稿数 | ✓ | - |
| 開設時期 | ✓ | 推定可 |
| 平均再生数 | ✓ | - |
| 最高再生投稿 | ✓ | URLと再生数 |
| エンゲージメント率 | - | いいね率、コメント率 |
| 投稿頻度 | - | 週◯本など |
| プロフィール文 | - | 全文 |
| 外部リンク | - | あれば |

### 1-2. 投稿サンプリング

5本を選定：
1. 最もバズった投稿（1本）
2. 平均的な投稿（2本）
3. 初期の投稿（1本）
4. 最新の投稿（1本）

各投稿について記録：投稿日時、再生数、動画の長さ、内容要約、音源、テロップスタイル

### 1-3. プロフィール3要素チェック

| 要素 | 確認ポイント |
|-----|-------------|
| **共感** | ターゲットの悩み・状況に共感しているか |
| **インセンティブ** | フォローする明確な理由があるか |
| **権威** | 信頼できる根拠・実績があるか |

**評価基準：** 3つ全てYes→◎ / 2つYes→○ / 1つ以下→△

---

## STEP 2: コンセプト分析（変数/定数の見極め）

### 2-1. 需要の特定

以下の質問に答える：
1. このジャンルの視聴者は何を知りたい？
2. どんな悩みを解決したい？（モテたい/痩せたい/稼ぎたい等）
3. どんな感情を求めている？（共感/カタルシス/代理体験/学び）

→ **需要A = 「_______________」**

### 2-2. 供給の分解

このジャンルの「よく見るフォーマット」を要素分解：

| 要素 | 内容例 |
|-----|-------|
| 出演者属性 | アラサーOL、大学生、主婦 |
| コンテンツ形式 | ナイトルーティン、レビュー、Vlog |
| 撮影方法 | 俯瞰、一人称、隠し撮り風 |
| 編集スタイル | 1秒1カット、スロー、タイムラプス |
| テロップ | 白フォント、丸ゴシック、手書き風 |
| 音声 | アフレコ、生音、BGMのみ |

### 2-3. 変数/定数の判定

各要素について3つの質問：
1. これを変えても需要Aに応えられる？
2. これを変えても本質は変わらない？
3. ジャンルの核心的要因ではない？

**判定：** 3つ全てYes→**変数**（変えてOK） / 1つでもNo→**定数**（変えてはダメ）

### 2-4. このアカウントの変数化ポイント

```
変数: [元の要素] → [このアカウントの変更点]
定数: [維持している成功要因]
```

**例：村上明子（60歳OL）**
- 変数: OL → 60歳おばあちゃん
- 定数: ナイトルーティン、Vlog形式、白フォント、アフレコなし

### 2-5. 成功/失敗の判定

- **成功パターン:** 変数のみ変更、定数は維持 → 差別化しつつ需要Aに応える
- **失敗パターン:** 定数を変えている → 架空の需要にしかヒットしない

## STEP 3: フォーマット特定

**詳細は [references/format-dictionary.md](references/format-dictionary.md) を参照**

### 主要フォーマットグループ

| グループ | 代表例 |
|---------|-------|
| **A: プロセスエコノミー型** | 起点/理想/過程の三角構造、やりたいことリスト型、他人軸プロセス |
| **B: ストリート・インタビュー型** | ストリート◯◯、タッチされたら◯◯、インタビュー |
| **C: Vlog・ルーティーン型** | 毎日◯◯生活、料理Vlog風、1秒1カットVlog |
| **D: ショートドラマ型** | 新感覚ショートドラマ、CSMモデル |
| **E: 特殊フォーマット** | 疑似恋愛3要素型、ネガティブフック型 |
| **F: 教育・有益系** | 3層構造モデル、半歩先アドバイス |

### 確認ポイント

1. どのフォーマットに該当するか？（複数可）
2. フォーマットの組み合わせはあるか？
3. オリジナル要素は何か？

## STEP 4: バズ要因の特定

**詳細は [references/psychology-triggers.md](references/psychology-triggers.md) を参照**

### 4-1. イントロ（冒頭3秒）分析

| カテゴリ | チェック項目 |
|---------|------------|
| **音的フック** | 効果音、強い言葉、ASMR |
| **映像的フック** | 強い動き、Before/After、意外性 |
| **言語的フック** | 問いかけ、ネガティブワード、数字、便益提示 |

**評価：** 3カテゴリ以上→◎ / 2つ→○ / 1つ以下→△

### 4-2. 心理トリガー

**ネガティブ系：** ネガティビティバイアス、リスク回避、シャーデンフロイデ
**ポジティブ系：** 好奇心、信頼性獲得
**その他：** バーナム効果、一貫性の法則、代理体験、疑似恋愛

### 4-3. エンゲージメント設計

- コメント余白型（応援/共感/ツッコミ/相談）
- コメント欄誘導型
- 炎上誘発型
- 質問形式

### 4-4. 継続視聴の設計

- シリーズ性（番号制、連続ドラマ）
- 焦らし（クリフハンガー）
- 二重構造（視覚×物語）
- 進捗の可視化

## STEP 5: マネタイズ設計の評価

### 5-1. 収益源の確認

| 収益源 | 有無 | 詳細 |
|-------|-----|------|
| 広告収益 | | プラットフォーム収益 |
| アフィリエイト | | 紹介商品: |
| PR案件 | | 案件内容: |
| 自社商品/サービス | | 販売内容: |
| その他 | | サロン、note等 |

### 5-2. 商品紹介の自然さ

- ◎ コンテンツの一部として自然
- ○ やや強引だが違和感なし
- △ 露骨すぎる
- × 導線なし（要改善）

### 5-3. 視聴者の購買力

| 要素 | 評価 |
|-----|------|
| 年齢層 | 10代以下（低）/ 20代（中）/ 30代以上（高） |
| 属性 | 学生（低）/ 社会人（中〜高）/ 主婦（中） |
| コメント欄の質 | 建設的（高）/ 普通（中）/ しょうもない（低） |

### 5-4. 「バズる ≠ 稼げる」チェック

| 状況 | 原因 | 対策 |
|-----|------|------|
| 再生取れるがフォロワー増えない | 保存価値がない | レシピ記載、有益性追加 |
| フォロワー多いが収益化できない | 視聴者の質が低い | ターゲット変更 |
| バズっているが思想がない | 応援されない | Who×What×How明確化 |

---

## STEP 6: 改善提案の作成

### 6-1. 良い点（3〜5個）

このアカウントの優れている点をリストアップ

### 6-2. 改善点の分類

**【致命的】（すぐ改善）**
- 思想がない（Who×What×How不明確）
- 定数を変えている
- マネタイズ導線が皆無
- 視聴者の質が極端に低い

**【重要】（優先度高）**
- フォロワー転換率が低い
- エンゲージメントが低い
- プロフィールが弱い
- イントロが弱い

**【改良余地】（優先度中）**
- 編集クオリティ
- Before/Afterの見せ方
- 投稿頻度
- コンテンツバリエーション

### 6-3. アクションプラン

| 優先度 | アクション | 期待効果 | 難易度 |
|-------|----------|---------|-------|
| 1（今すぐ） | | | |
| 2（1週間以内） | | | |
| 3（1ヶ月以内） | | | |

### 6-4. 横展開・応用提案

このアカウントの成功要因を他ジャンルに応用する案を提示：
- ジャンル:
- 変数変更:
- 定数維持:
