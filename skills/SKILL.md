## STEP 2: コンセプト分析（変数/定数の見極め）

### 2-1. ターゲット設定と需要の特定

#### ターゲット設定の原則

**「架空のペルソナ」を作るな。「実在する1人」をターゲットにしろ。**

| NGパターン | なぜダメか |
|-----------|----------|
| 「20代〜40代の女性」 | 広すぎてコンテンツがブレる |
| 「架空のA子さん（28歳、IT企業勤務...）」 | 都合の良い幻想を抱きがち。そんな人は実在しない |

**正しいターゲット設定：** 実在する1人を選ぶ

| 候補 | 例 |
|-----|---|
| 過去の自分 | 「1年前の自分が知りたかったこと」 |
| 職場の同僚 | 「隣の席のあの人が悩んでいること」 |
| 身内の人間 | 「姉が困っていたあの問題」 |
| 地元の友人 | 「あいつなら絶対こういうの見るだろう」 |

目的達成のために「誰を集めたいか」から考え、その実在する人がSNSを使っている前提で設計する。

#### 逆ペルソナ設計（レコメンド型SNS特有の戦略）

TikTok・インスタリールなどレコメンド型SNSでは、フォロワー以外にも動画が配信される。そのため従来の「見込み客をペルソナに設定する」やり方では、見込み客以外の大多数が離脱し、アルゴリズム評価が下がる。

**原則：「絶対に来ない人」をペルソナに設定し、全員が離脱しない動画を作る**

| 従来の考え方 | レコメンド型SNSの考え方 |
|------------|-------------------|
| 見込み客だけに刺さる動画を作る | 見込み客以外も含めて「誰が見ても離脱しない」動画を作る |
| ペルソナ = 理想の顧客像 | ペルソナ = 最も自分のサービスに興味がない人 |
| ターゲット外の離脱は気にしない | ターゲット外が離脱するとアルゴリズム評価が下がる |

**実装方法：**
1. 自分のサービスに「絶対に来ない層」を特定する（例：ホテル事業なら「旅行に興味がない在宅ワーカー」）
2. その人でも最後まで見てしまう動画を設計する
3. 結果として見込み客にも当然刺さり、かつアルゴリズム評価も高くなる

**注意：** これは「万人ウケの薄い動画を作れ」という意味ではない。エンタメ性や情報価値で全員を引きつけつつ、本来のターゲットにはさらに深く刺さる二層構造にする。

#### スクリーンレコード法（ターゲット理解の最強手法）

ターゲット像の精度を上げるための具体的リサーチ手法：

1. ターゲットにスマホを渡し「5分間好きにSNSを見て」とお願いする
2. その画面を録画させてもらい、観察する

**分かること：**
- タイムラインに何が流れているか（リズム・画角・ジャンル）
- どこを触っているか（どの動画で止まるか、スキップするか）
- どういう使い方をしているか

**重要な発見：** 自分の想定と全く違う世界が見える（例：日本語が流れてこない、文字動画を見ない等）。ターゲットが実際にどんなコンテンツに囲まれているかを理解してから設計に入る。

#### 需要Aの特定

以下の質問に答える：
1. このジャンルの視聴者は何を知りたい？
2. どんな悩みを解決したい？（モテたい/痩せたい/稼ぎたい等）
3. どんな感情を求めている？（共感/カタルシス/代理体験/学び）

→ **需要A = 「_______________」**

### 2-2. 供給の分解

このジャンルの「よく見るフォーマット」を要素分解：

| 要素 | 内容例 |
|-----|-------|
| 出演者属性 | アラサーOL、大学生、主婦 |
| コンテンツ形式 | ナイトルーティン、レビュー、Vlog |
| 撮影方法 | 俯瞰、一人称、隠し撮り風 |
| 編集スタイル | 1秒1カット、スロー、タイムラプス |
| テロップ | 白フォント、丸ゴシック、手書き風 |
| 音声 | アフレコ、生音、BGMのみ |

### 2-3. 変数/定数の判定

各要素について3つの質問：
1. これを変えても需要Aに応えられる？
2. これを変えても本質は変わらない？
3. ジャンルの核心的要因ではない？

**判定：** 3つ全てYes→**変数**（変えてOK） / 1つでもNo→**定数**（変えてはダメ）

### 2-4. このアカウントの変数化ポイント

```
変数: [元の要素] → [このアカウントの変更点]
定数: [維持している成功要因]
```

**例：村上明子（60歳OL）**
- 変数: OL → 60歳おばあちゃん
- 定数: ナイトルーティン、Vlog形式、白フォント、アフレコなし

### 2-5. 成功/失敗の判定

- **成功パターン:** 変数のみ変更、定数は維持 → 差別化しつつ需要Aに応える
- **失敗パターン:** 定数を変えている → 架空の需要にしかヒットしない

## STEP 3: フォーマット特定

**詳細は [references/format-dictionary.md](references/format-dictionary.md) を参照**

### 主要フォーマットグループ

| グループ | 代表例 |
|---------|-------|
| **A: プロセスエコノミー型** | 起点/理想/過程の三角構造、やりたいことリスト型、他人軸プロセス |
| **B: ストリート・インタビュー型** | ストリート◯◯、タッチされたら◯◯、インタビュー |
| **C: Vlog・ルーティーン型** | 毎日◯◯生活、料理Vlog風、1秒1カットVlog |
| **D: ショートドラマ型** | 新感覚ショートドラマ、CSMモデル |
| **E: 特殊フォーマット** | 疑似恋愛3要素型、ネガティブフック型 |
| **F: 教育・有益系** | 3層構造モデル、半歩先アドバイス |
| **G: バズ構文型** | 稀有な人×普通質問構文、逆張り構文、プロセス×ストリート融合 |

### リズム同調の法則（フォーマット選定の大前提）

**自分の畑（タイムライン）で戦え。**

ショート動画は「心地よいリズム」が全て。自分が日常的に見ているタイムラインに流れていないフォーマットを採用しても、リズム感覚がチューニングできず失敗する。

| 原則 | 説明 |
|-----|------|
| Vlogを見ない人がVlogをやっても伸びない | リズム感覚が身についていないため品質判断ができない |
| トークを見ない人がトークをやっても伸びない | 間の取り方、テンポ感が体感で分からない |
| 自分が日常的に視聴しているフォーマットを採用するのが最も再現性が高い | リズムの良し悪しを自分の感覚で判断でき、PDCAの精度が上がる |

**フォーマット選定時の確認：** このフォーマットは自分（またはクライアントの運用担当者）が日常的に視聴しているか？ NOなら、まずそのフォーマットを100本見てリズムを体に染み込ませるか、別のフォーマットを選ぶ。

### フォーマットと信頼感の関係（HPゲージへの影響）

フォーマット選定は「リズム同調」だけでなく、視聴者側の信頼感にも直結する。見慣れたフォーマットはHPゲージにボーナスを与え、見慣れないフォーマットは初期HPを大幅に削る：

| 状況 | HPゲージへの影響 |
|-----|---------------|
| 視聴者が普段見慣れたフォーマット | 「ちゃんとした動画だ」と無意識に信頼→初期HPが高い状態でスタート |
| 視聴者が見たことないフォーマット | 「なんだこれ？素人か？」と不安→初期HPが低い状態でスタート |
| ジャンル内で定番化しているフォーマット | そのジャンルの視聴者にとって「正解の型」として認知されている→安心感 |

**つまり：** フォーマットは「テンプレ＝悪」ではない。伸びているフォーマットを採用すること自体が視聴者への信頼のシグナルになる。その上で変数/定数理論で差別化する。

### 確認ポイント

1. どのフォーマットに該当するか？（複数可）
2. フォーマットの組み合わせはあるか？
3. オリジナル要素は何か？
4. **そのフォーマットは運用者が日常的に視聴しているか？**（リズム同調チェック）

## STEP 4: バズ要因の特定

**詳細は [references/psychology-triggers.md](references/psychology-triggers.md) を参照**

### 4-0. 動画分析の2指標（内容と構成を分けて評価する）

動画のパフォーマンスを振り返る際、「内容（ネタ）」と「構成（見せ方）」は別々に評価する。両方同時に変えると何が原因か分からなくなる。

#### 指標①：内容の評価（初速エンゲージメント率）

| 項目 | 内容 |
|-----|------|
| **見る指標** | 再生回数に対するエンゲージメント率（いいね・保存など） |
| **タイミング** | 投稿直後（初速）。バズって外部に広がると率は下がるため、最初の24〜48時間で判断 |
| **基準値** | アカウントごとの平均エンゲージメント率を基準に設定（例：2%が基準なら、2%超え＝内容良い、2%未満＝内容悪い） |
| **判断** | 初速で反応が悪ければ「内容（ネタ）」が悪かったと振り返る |

#### 指標②：構成の評価（冒頭3秒の離脱率）

| 項目 | 内容 |
|-----|------|
| **見る指標** | インサイトの維持率グラフ（TikTok/インスタのスキップ率など） |
| **タイミング** | 投稿後1〜3時間で数字が安定した頃 |
| **基準値** | 冒頭3秒維持率 50%以上→合格 / 30%未満→即諦める（その動画は伸びない） |
| **改善点** | 冒頭3秒は「映像（コピー含む）」と「音」で決まる |

#### 使い分けの考え方

```
内容（ネタ）が良い × 構成が良い → バズる
内容（ネタ）が良い × 構成が悪い → 冒頭で離脱されて伸びない。構成を変えて再投稿の価値あり
内容（ネタ）が悪い × 構成が良い → 見てはもらえるがエンゲージメントが低い。ネタを変える
内容（ネタ）が悪い × 構成が悪い → 全体的にやり直し
```

### 4-1. イントロ（冒頭3秒）分析

| カテゴリ | チェック項目 |
|---------|------------|
| **音的フック** | 効果音、強い言葉、ASMR |
| **映像的フック** | 強い動き、Before/After、意外性、しずる感（肉汁・チーズ等の本能訴求） |
| **言語的フック** | 問いかけ、ネガティブワード、数字、便益提示 |

**評価：** 3カテゴリ以上→◎ / 2つ→○ / 1つ以下→△

**冒頭2段階フック設計（1投稿1万フォロワー事例から抽出）：**
1. **1秒目：止める**（未回収の疑問を置く）→「何が起きた？」「え、どういうこと？」
2. **2秒目：引き込む**（数字付きベネフィットを提示）→「見たい」「自分に関係ある」に変える

**追加チェック：**
- 権威性×常識破壊の組み合わせはあるか？（「20年のベテランが言う→実は○○は不要」）
- テロップは13文字以内で瞬間理解できるか？（長文テロップ＝離脱要因）
- 動画冒頭とアカウントの世界観は一致しているか？（一致→フォロー転換率UP）

### 4-1b. 気まずいメーター（編集・構成の設計原則）

**編集は「凄さ」を見せるためではなく、「気まずさ」を解消するためにある。**

#### 基本概念：HPゲージモデル

視聴者は全員、無意識レベルで「動画を見続けるか飛ばすか」を常に判定している。これをHPゲージ（気まずいメーター）で可視化する：

- HPゲージは時間経過とともに常に減少し続ける
- HPが0になる＝スワイプされる（次の動画に飛ばされる）
- 動画中に「面白い」と感じるポイントがあるとHPが回復する
- 0になる前に回復させ続けることで視聴を維持する

**視聴者の深層心理：** 「時間をかけて見たのに面白くなかったを、とにかく避けたい」＝時間を無駄にしたくない。タイパ（タイムパフォーマンス）の時代において、この感覚は年々強まっている。

#### 対処法

メーターが0になる直前にSE（効果音）を「パンッ」と入れて脳を刺激し、メーターを10まで回復させる。

#### 冒頭で即死するパターン（HPが一瞬で0になる要因）

以下に当てはまる動画は冒頭1〜2秒で飛ばされる。共通点は「何かしらのストレスがある」こと：

| パターン | 詳細 |
|---------|------|
| 画質が悪い | 画質が悪い→見続ける価値がない、と一瞬で判断される |
| 音が悪い | 音割れ、音量が大きい/小さい、AI読み上げ感が強い |
| 視線が散る | 情報量が多く、どこを見たらいいか迷う＝ストレス。文字数過多、テロップが離れすぎ、色がごちゃごちゃ |
| テロップ視認性が悪い | 背景とテロップの色が近くて見にくい。「見れないこともないけど見にくい」でも飛ばされる |
| 目がチカチカする | 無駄に明るい配色、映像切り替わりが速すぎて全部明るい |
| ノイズが多い | 画面内に意図のないもの（チラシ、生活感のあるボトル等）が映り込み→プロ感の欠如→信頼低下 |

#### 冒頭以降で死ぬパターン（HPがじわじわ削られる要因）

| パターン | 詳細 |
|---------|------|
| **テンポが悪い** | セリフ間の「意味のない間」が0.5秒あるだけでストレス。伸びているアカウントはセリフ間が長くても0.2秒。無駄な間は徹底的にカットする |
| **ナレーションが遅い** | ゆっくり話している間にもHPは減り続ける。視聴者が理解できる範囲で早く話す方が良い。尺が足りないなら台本（情報量）を増やすのが正解 |
| **画面に動きがない** | 映像的に静止している動画はHPの減りが劇的に早い。常に何かが動いている映像を心がける |
| **無駄なことを言っている** | HPが回復しない言葉を発している時間は全て損失 |

#### 「無駄なこと」の具体例（全て不要）

| やりがちなこと | なぜダメか |
|-------------|----------|
| 挨拶（「はいどうも皆さん」） | 動画の最も大事な冒頭部分を犠牲にしている。ショート動画で挨拶を求めている人は0 |
| 「いいね・保存してね」 | 心理的リアクタンス（押しつけへの反発）が働き逆効果。面白ければ勝手にいいねする |
| 「共有から開いて○番目の人は〜」 | エンゲージメントハックの魂胆が見え透いて嫌われる |
| 「最後のはやばい」 | 無駄にHPを減らしているだけ |
| 「気になる1位はコメント欄で」 | 「スっと言えよ」と思われて終わり |

**原則：** 視聴者に何かしてほしい行動があるなら、直接的に依頼するのではなく、そうしたくなるように自然と導く。動画が面白ければエンゲージメントは勝手についてくる。

#### 実践ポイント

- 高度な編集スキルは不要（ジェットカット等の基礎で十分）
- 凝った編集よりも「何を言うか（内容）」が重要
- 離脱率が悪い時は映像だけでなく「音（BGM・SE）」に問題がある可能性が高い
- 音は脳（本能）に直接働きかけるため、音が合っていれば中身を見なくても進行させられる
- テロップは「注目させたい部分に視線を集める」ことを最優先に設計する

### 4-2. 心理トリガー

**ネガティブ系：** ネガティビティバイアス、リスク回避、シャーデンフロイデ
**ポジティブ系：** 好奇心、信頼性獲得
**その他：** バーナム効果、一貫性の法則、代理体験、疑似恋愛

#### 伸びる動画の6パターン（視聴者の心理を動かす仕掛け）

| パターン | 原理 | 実装のポイント |
|---------|------|-------------|
| **①自分事化** | 「自分に関係ある」と思った瞬間に興味が生まれる | 「〇〇な人は見て」「〇〇してない？」等、ターゲットが「自分のことだ」と感じるフックを冒頭に置く。最も汎用的で強力なパターン |
| **②プロスペクト理論（損失回避）** | 人は「得すること」より「損すること」に2倍強く反応する | 「知らないと損する〇〇」「〇〇してる人、やばいです」等、損失を示唆する。ネガティブフックはこの原理 |
| **③ヒキのある映像** | 言語処理より先に視覚が反応する（本能的に目が止まる） | 5パターン：自然現象（炎、水しぶき等）、満足感（きれいに剥がれる等）、本能（食欲・性欲に訴える）、心霊・超常現象、超絶技巧。映像自体の力で3秒の壁を突破する |
| **④ホットな話題** | 今まさに世間が関心を持っている話題は検索・表示されやすい | トレンドワード、時事ネタ、季節イベントに自分のジャンルを掛け合わせる。鮮度が命なので速度重視 |
| **⑤議論を呼ぶ** | 意見が割れるテーマはコメント欄が活性化→アルゴリズム優遇 | 「〇〇って実はこうじゃない？」等の問題提起。ただし炎上リスクあり、ジャンルによって使い分ける |
| **⑥スカッと・カタルシス** | 溜まったストレスが解消される快感＝強い視聴動機 | 論破系、復讐系、掃除のビフォーアフター、害虫駆除等。視聴者が日常で感じている不満を代理で解消する |

**テーマ作成の実践法：上の句/下の句メソッド**

動画テーマを作る際、「自分事化フック（上の句）」＋「内容（下の句）」の2段構成で考える：

| 構成 | 役割 | 例 |
|-----|------|---|
| **上の句** | 自分事化する入口。ターゲットが「私のことだ」と思うフレーズ | 「営業マンなら共感する」「一人暮らしの人、これ知ってる？」 |
| **下の句** | 実際に伝える内容。行動やノウハウ、結果 | 「〇〇の使い方」「〇〇したら人生変わった」 |

例：「一人暮らしの人、これ知ってる？」（上の句）＋「業務スーパーの冷凍食品ベスト5」（下の句）

### 4-2b. 信頼の構造設計（顔出しなしでも勝てる仕組み）

顔出しできない場合でも「信頼」を構造的に作る方法：

| 手法 | 仕組み |
|-----|-------|
| **第三者リアクション** | 自称「美味しい」ではなく、第三者が本気で驚く声/反応を入れる→信頼性が跳ね上がる |
| **想像の余白** | 顔を隠すことで視聴者が自分の状況に置き換えて想像できる→保存率・フォロー率UP |
| **視覚の統一** | 画面に映る空間全体がアカウントの「顔」になる。色・素材・トーンを統一し、意図のないものは1ミリも映さない |

### 4-3. エンゲージメント設計

- コメント余白型（応援/共感/ツッコミ/相談）
- コメント欄誘導型
- 炎上誘発型
- 質問形式

#### コメント誘発テクニック（意図的にコメントを引き出す仕掛け）

コメント数はアルゴリズム上最も重要な指標の一つ。ただし「コメントしてね」と直接言うのは心理的リアクタンス（反発）を招くため逆効果。以下の手法で「自然とコメントしたくなる」設計にする：

| テクニック | 仕組み | 例 |
|-----------|-------|---|
| **ツッコミ待ち** | わざと間違える・噛む・変なことを言い、視聴者が指摘したくなる隙を作る | 料理動画で明らかに調味料の量を間違える、言い間違いをそのまま残す |
| **論理矛盾** | 動画内であえて矛盾する主張をして「え、さっきと違くない？」と書き込ませる | 「絶対やるな」と言った後に自分がやっている映像を入れる |
| **元ネタ指摘** | 誰もが知っている曲・映画・ネタのオマージュを入れ、「あ、これ○○じゃん」と書き込ませる | BGMや構図に有名作品のオマージュを仕込む |
| **議論誘発** | 意見が割れるテーマを提示し、賛否両論のコメントを呼ぶ | 「○○と△△、どっちが正しい？」系のテーマ設定 |
| **診断・あるある** | 「あなたは○○タイプ」「○○な人あるある」で自分のことを語りたくなる | 「やっぱりBだった」「これ完全に自分」等のコメントが発生 |

**重要原則：** 心理的リアクタンスを避ける。「いいね・保存してね」「フォローしてね」等の直接的な依頼は逆効果。面白い動画を作れば、エンゲージメントは自然についてくる。行動を促したいなら、直接言うのではなく「そうしたくなるように設計する」。
