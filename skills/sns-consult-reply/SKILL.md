# Discord SNS相談返信システム — SKILL定義

## 概要
Discordに来るSNS運用相談を、ユーザー（運営者）のノウハウを活かして返信する仕組み。
相手をウザがらせず、長文で返しすぎず、でもクリティカルな助言を返す。

---

## 処理フロー

```
1. 相談受信 → 内容を読む
2. 分類 → 6分類のどれに当たるか判定
3. 情報判定 → 返信に十分な情報があるか判定
   ├─ 十分 → 4へ
   └─ 不十分 → 最小ヒアリング（3〜5問）→ 回答受領後に4へ
4. 内部分析 → SKILLフレームワーク + referencesを使って分析
5. 返信生成 → 短く刺す返信を生成
6. 承認 → ユーザーに送信内容を確認（外部送信のため承認必要）
7. 送信
```

---

## 6分類

相談は以下の6カテゴリに分類する：

### 1. アカウント設計相談
- 「何から始めればいい？」「コンセプトが決まらない」
- 使うリファレンス: SKILL.md全体、format-dictionary.md

### 2. 伸び悩み相談
- 「フォロワーが増えない」「再生数が落ちた」
- 使うリファレンス: failure-patterns.md、operation-strategy.md

### 3. コンテンツ企画相談
- 「ネタが尽きた」「何を投稿すればいい？」
- 使うリファレンス: script-creation.md、format-dictionary.md

### 4. バズ・エンゲージメント相談
- 「バズりたい」「コメントが来ない」
- 使うリファレンス: psychology-triggers.md、case-studies.md

### 5. 収益化相談
- 「フォロワーいるけど稼げない」「案件が来ない」
- 使うリファレンス: SKILL.md（収益化セクション）、failure-patterns.md

### 6. プラットフォーム固有相談
- 「TikTokとInstagramどっちがいい？」「アルゴリズムがわからない」
- 使うリファレンス: operation-strategy.md

## 情報判定基準

返信に最低限必要な情報：

| 分類 | 必須情報 |
|------|---------|
| アカウント設計 | ジャンル、ターゲット、現在の状態 |
| 伸び悩み | アカウントURL or スクショ、現在のフォロワー数、投稿頻度 |
| コンテンツ企画 | ジャンル、過去にウケた投稿、ターゲット層 |
| バズ | 直近の投稿内容、再生数の推移 |
| 収益化 | フォロワー数、ジャンル、現在の収益手段 |
| プラットフォーム | 使用プラットフォーム、ジャンル、目的 |

不足情報がある場合のみヒアリングを行う。

---

## 最小ヒアリング設計

### 原則
- 最大5問（3問が理想）
- 1問1行で聞く
- 選択式にできるものは選択式にする
- 「なんでも教えてください」は禁止（相手が疲れる）

### テンプレート
```
ありがとう！いくつか聞かせて：
1. [具体的な質問]
2. [具体的な質問]
3. [具体的な質問]
わかる範囲でOK！
```

---

## 内部分析プロセス

1. 分類に対応するリファレンスファイルを読む
2. 相談内容と照合し、該当するパターンを特定
3. case-studies.mdから類似事例を探す
4. failure-patterns.mdで該当する失敗パターンがないか確認
5. 具体的なアクション提案を3つ以内に絞る

---

## 返信生成ルール

### トーン
- フランク（ですます調でなくていい）
- 短い（200文字以内が理想、最大400文字）
- 具体的（「頑張りましょう」みたいな抽象論は禁止）

### 構造
```
[共感/承認] 1行
[核心の指摘] 1-2行
[具体アクション] 1-3個（箇条書き）
[補足/励まし] 1行（任意）
```

### 禁止事項
- 長文で返すこと（相手が読まない）
- 専門用語を並べること（相手は初心者の可能性がある）
- 「色々ありますが〜」的な曖昧回答
- 質問攻めにすること（ヒアリングは最小限）
- 上から目線（相手は相談してくれている）

### 良い返信例
```
おー、料理系いいね。
正直今のままだと「レシピ動画」として埋もれやすい。
差別化ポイント作ろう：
- 冒頭2秒で「完成品のうまそうな画」を見せる
- 「誰が作ってるか」を出す（顔じゃなくても手元+声でOK）
- 投稿タイトルは「レシピ名」じゃなくて「食べた人の反応」にする
これだけで保存率変わるから試してみて。
```

### 悪い返信例
```
ご相談ありがとうございます。
料理系アカウントの運用について、いくつかの観点からアドバイスさせていただきます。
まず、コンテンツの差別化についてですが、現在の市場においては...
（以下3000文字続く）
```

---

## 承認フロー

返信を送信する前に、必ずユーザー（運営者）に確認する：

```
[Discord返信ドラフト]
相手: @username
分類: 伸び悩み相談
---
（返信内容）
---
送信OK？
```

これは外部送信に該当するため、TOOLS.mdの承認ルールに従う。
