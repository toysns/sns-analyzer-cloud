# 台本・テーマ作成ガイド

動画のテーマ選定から台本作成までの実践的な手法をまとめる。

---

## 1. テーマ作成法

### 1-1. 上の句/下の句メソッド

動画テーマは「自分事化フック（上の句）」＋「伝えたい内容（下の句）」の2段構成で考える。

| 構成 | 役割 | 機能 |
|-----|------|------|
| **上の句** | 自分事化する入口 | ターゲットが「自分のことだ」と感じるフレーズ。ここで止まるかスワイプされるか決まる |
| **下の句** | 実際に伝える内容 | 行動、ノウハウ、結果、価値提供 |

#### 上の句のパターン

| パターン | 例 |
|---------|---|
| 属性指定 | 「営業マンなら共感する」「一人暮らしの人」 |
| 行動指定 | 「まだ○○してる人」「○○してない？」 |
| 状況指定 | 「彼氏にフラれた時」「上司に怒られた帰り道」 |
| 感情指定 | 「やる気が出ない日」「将来が不安な人」 |

#### 下の句のパターン

| パターン | 例 |
|---------|---|
| ノウハウ | 「○○の使い方」「○○で解決する方法」 |
| ランキング | 「○○ベスト5」「絶対やるべき○○3選」 |
| 体験談 | 「○○したら人生変わった」「○○してみた結果」 |
| 比較 | 「○○ vs ○○、どっちが正解？」 |

### 1-2. テーマ量産の考え方

上の句と下の句を別々にストックし、組み合わせでテーマを量産する。

```
上の句ストック × 下の句ストック ＝ 大量のテーマ候補
```

同じ下の句（内容）でも上の句（自分事化フック）を変えるだけで別の動画になる。例：
- 「一人暮らしの人」× 業務スーパーベスト5
- 「自炊がめんどくさい人」× 業務スーパーベスト5
- 「食費を月2万に抑えたい人」× 業務スーパーベスト5

## 2. 台本作成の3原則

### 原則①：聞いて理解できること

- 「読めば分かる」はダメ。「聞いただけで分かる」が必須
- 難しい漢字や専門用語は口語に置き換える
- 一文は短く。「〜で、〜で、〜なので、〜です」のような長い文は禁止
- 台本を書いたら必ず声に出して読む

### 原則②：引き算の美学

- 情報は「足す」のではなく「削る」
- 1動画1メッセージが基本。あれもこれも言わない
- 「この一文がなくても意味が通るか？」→通るなら削除
- 尺が足りないからと情報を薄めて引き伸ばすのは最悪（HPゲージが即死する）

### 原則③：結論ファースト

- 冒頭で「この動画を見ると何が得られるか」を明示
- 前置きが長い動画は離脱される
- 「まず結論から言うと〜」で始め、その後に根拠・理由を展開

## 3. 台本構造パターン

### パターンA：問題提起型（教育・有益系向き）

```
1. 問題提起（自分事化フック）
2. 結論（先に答えを言う）
3. 理由・根拠
4. 具体例
5. まとめ（結論の繰り返し）
```

### パターンB：ストーリー型（体験談・Vlog系向き）

```
1. 結果の提示（冒頭で最も衝撃的な結果を見せる）
2. 背景（なぜそうなったか）
3. 行動（何をしたか）
4. 転機（どこで変わったか）
5. 現在（今どうなっているか）
```

### パターンC：ランキング型（情報系向き）

```
1. テーマ提示（「○○ベスト5」）
2. 5位〜2位（テンポよく）
3. 1位（最も厚く解説）
4. まとめ
```

注意：ランキングは下位から。1位を先に言うと残りを見る理由がなくなる。

### パターンD：比較型（議論系向き）

```
1. 問い（「○○と△△、どっちが正解？」）
2. Aの主張と根拠
3. Bの主張と根拠
4. 自分の結論
5. コメント誘発（「みんなはどっち？」）
```

## 4. ロジック作成法（「なぜ？」の連鎖で説得力を作る）

台本で最も難しいのは「なぜそう言えるのか」の論理構築。以下の手法で連想的にロジックを作る：

### 手順

1. **主張を決める**（例：「朝活は最強」）
2. **「なぜ？」を3回繰り返す**
   - なぜ朝活が最強？→ 集中力が高い
   - なぜ集中力が高い？→ 脳のエネルギーが満タン
   - なぜ朝だとエネルギーが満タン？→ 睡眠で回復しているから
3. **逆に「しないとどうなる？」を考える**（損失回避の心理を使う）
   - 朝活しないと？→ 夜にやる→疲れてて効率悪い→結局やらない→自己嫌悪
4. **権威・データで補強する**
   - 「ハーバード大学の研究で〜」「統計によると〜」

### ポイント

- 完璧な論理は不要。視聴者が「なるほど」と感じる程度でOK
- ロジックは「正しさ」より「納得感」が重要
- 難しい理屈より身近な例え話の方が刺さる

## 5. 冒頭3秒の台本設計

冒頭3秒は台本上最も重要。以下のいずれかで始める：

| 手法 | 例 | 心理原理 |
|-----|---|---------|
| 損失回避 | 「これ知らない人、損してます」 | プロスペクト理論 |
| 自分事化 | 「○○な人、ちょっと来て」 | 関連性 |
| 衝撃の事実 | 「実は○○って嘘なんです」 | 好奇心ギャップ |
| 結果先出し | 「これやったら売上3倍になりました」 | 具体性 |
| 否定 | 「○○は絶対やめてください」 | ネガティビティバイアス |
| しずる感 | 肉汁が溢れる、チーズが伸びる等の本能訴求映像 | 動物的本能（食欲・五感） |

**NG例：**
- 挨拶から始める（「はいどうも〜」）
- 前置きが長い（「今日はですね〜」）
- 何の動画か分からない冒頭

## 6. キャッチフレーズの3構造（冒頭の自己紹介を進化させる）

初心者は冒頭で「○○です。今日は〜」と自己紹介しがち。これを「キャッチフレーズ導入」に進化させる手法：

### 自己紹介 → キャッチフレーズ への進化

| 段階 | 冒頭の言葉 | 効果 |
|-----|----------|------|
| 初心者 | 「ど素人ホテル再建計画です」 | 名前を知らなければスワイプされる |
| 中級者 | 「ゼロからホテルを作ってます」 | 何をしている人か分かるがフックが弱い |
| 上級者 | 「怪しい市場をぶっ壊す男です」 | 感情を動かす＋何者か気になる→視聴継続 |

### キャッチフレーズの3構造

| ステップ | 機能 | 例 |
|---------|------|---|
| ①感情を動かす強ワード | 冒頭で「え？」と思わせる言葉 | 「怪しい市場をぶっ壊す」 |
| ②事実に紐づけ | 強ワードが嘘でないことを示す根拠 | 実際に怪しい市場（ホテル運営代行）で透明な料金体系を導入している事実 |
| ③本文で伏線回収 | 動画の内容が①の意味を深める | 動画内で実際に「怪しい業者」と交渉する場面を見せる |

**原則：** ①が事実に基づかない（盛りすぎ・嘘）場合、一時的にバズっても信頼を失う。「強ワードだが本当のこと」が最強。

## 7. 「誤解→共感→回収」構造（プロセスエコノミー台本の黄金法則）

プロセスエコノミー型コンテンツにおける台本の鉄板構造：

### 3ステップ

| ステップ | 視聴者の心理 | 台本上の動き |
|---------|-----------|-----------|
| ①誤解 | 「え、この人何言ってるの？」「それは違うでしょ」 | あえて誤解を招く発言や行動を冒頭に置く。ツッコミコメント余白が生まれる |
| ②共感 | 「あ、なるほど、そういう背景があったのか」 | 誤解の背景にある事情・理由を丁寧に説明。感情的に寄り添う |
| ③回収 | 「この人すごいかも」「応援したい」 | ①の誤解が実は正しかった/意味があったことが判明。信頼と応援に変わる |

### 例

```
①誤解：「ホテルの支配人をクビにしました」（え？ひどくない？）
②共感：「実は3ヶ月間、改善を提案し続けたけど聞いてもらえなかった」（そんな背景が...）
③回収：「新しい支配人の下で売上が200%になった」（判断は正しかったんだ→すごい）
```

**効果：** コメント欄に「最初は嫌な人かと思ったけど」「最後まで見てよかった」等のコメントが増える→エンゲージメント↑→アルゴリズム評価↑

## 8. 冒頭2段階フック設計（1秒で止める＋2秒で引き込む）

冒頭3秒を「1秒」と「2秒」に分解し、それぞれ異なる役割を持たせる。1万フォロワー超え事例に共通するパターン：

### 第1段階：1秒で止める（スクロール停止）

自動再生の中でスクロールを物理的に止めさせる。ここで止まらなければ全てが無意味。

| 手法 | メカニズム | 例 |
|-----|----------|---|
| **視覚的異常** | 普通の画面と明らかに違う映像で目を止める | 全身コーデが1色統一、巨大な料理、異常な量の商品 |
| **テロップ衝撃** | 13文字以内の短いテロップで「え？」と思わせる | 「これ全部100均」「月収3倍になった」 |
| **しずる感** | 食欲・五感に直接訴える本能的映像 | 肉汁、チーズとろり、湯気、断面 |
| **美的圧倒** | 圧倒的に美しい/かっこいい映像で目を奪う | プロ級の盛り付け、完璧なインテリア |

### 第2段階：2秒で引き込む（視聴継続の決定）

止まった視聴者に「最後まで見たい」と思わせる。ここでフォロー動機の種も撒く。

| 手法 | メカニズム | 例 |
|-----|----------|---|
| **権威性×常識破壊** | まず信頼させてから常識を壊す→「なぜ？」が生まれる | 「管理栄養士が教える、実は体に悪い健康食品」 |
| **結果の先出し** | 結果を見せて「どうやって？」という疑問を作る | Before/After映像、数字の変化 |
| **謎の提示** | 答えが気になる問いを投げる | 「この部屋、実は○万円で作れます」 |
| **共感からの展開予告** | 共感で引き込み、解決策があることを匂わせる | 「○○で悩んでた私が、たった1つのことで解決した」 |

### テロップの13文字ルール

自動再生で一瞬で理解させるテロップは**13文字以内**が限界。

| 文字数 | 認知速度 | 適用場面 |
|-------|---------|---------|
| 7文字以内 | 0.5秒で理解 | 最強。冒頭テロップに最適 |
| 8〜13文字 | 1秒で理解 | 冒頭テロップの上限 |
| 14文字以上 | 読む必要がある | 冒頭には不向き。本文テロップ向け |

**原則：** 冒頭テロップは「読ませる」のではなく「見た瞬間に入る」もの。文字数を削って削って削り切る。

## 9. 保存される台本設計（保存率を意識した構造）

「いいね」は感情、「保存」は実用。保存率が高い動画はアルゴリズムに強く評価される。

### 保存を誘発する台本構造

| 要素 | 設計方法 | 例 |
|-----|---------|---|
| **情報密度を高める** | 1動画に「あとで見返したい」レベルの情報を詰める | レシピの分量、手順、コツを全て網羅 |
| **リスト構造** | 「○選」「○つのコツ」等、まとめ形式にする | 「買ってよかった100均グッズ7選」 |
| **再現性を明示** | 「自分もやれそう」と思わせる具体的手順 | 材料名、価格、購入場所まで記載 |
| **テロップに情報を集約** | 音を消しても内容が分かるテロップ設計 | 手順番号、分量、ポイントをテロップ化 |

### 保存を促すCTA（Call To Action）

台本の最後に自然に保存を促す一言を入れる：
- 「忘れないように保存しておいてね」
- 「買い物の時に見返せるように保存推奨」
- テロップで「📌保存して見返してね」

**注意：** 保存CTAは「なぜ保存すべきか」の理由とセットで伝えると効果的。理由なき「保存してね」は効かない。

## 10. 伸びた型を擦り続ける（シリーズ台本設計）

1本バズった構造は、内容を変えて繰り返し使う。「飽きられるのでは？」は杞憂。視聴者の大多数は初見。

### 擦り方の原則

| 原則 | 説明 |
|-----|------|
| **構造は固定、中身だけ変える** | テロップの出し方、カット割り、BGM、テンポは同じ。紹介するアイテムや情報だけ変える |
| **サムネ・冒頭の型を統一** | シリーズ感を出すことで「このアカウントの動画だ」と認知される→フォロー率UP |
| **数字で管理する** | 伸びなくなったら初めて型を変える。感覚ではなくデータで判断 |
| **世界観を崩さない** | フォント、色、トーン、BGMを統一。1本だけ違う雰囲気の動画を出すとフォロワーが混乱する |

### シリーズ導線の台本設計

シリーズ内で次の動画への導線を台本に組み込む：

```
パターン①：予告型
「次回は○○編。フォローして見逃さないでね」

パターン②：伏線型
「実はこの中で1つだけ注意点があるんだけど、それは次の動画で」

パターン③：リクエスト型
「コメントで次にやってほしいテーマ教えて」（→コメント数UP＋次回ネタ収集）
```

**本質：** 台本は「1本で完結させる」のではなく「次の動画を見たくなる仕掛け」を常に含める。これがフォロー率を決定的に変える。
