# TOOLS.md — ツール固有の使い方

> 汎用ルール（停止条件、報告ルール等）は **RULES.md** を参照。ここにはツール固有の情報だけ書く。

---

## Slack送信

### 承認レベル
- **実況チャンネルへの送信**: 承認不要（常時許可）
- **外部の第三者への送信**: 承認必要

### 使い方
- 能動的に使う（ユーザーのメッセージを待たない）
- 途中でも送る。まとめずに送る。今ある情報で送る。

---

## ファイル操作

| 操作 | 承認 |
|------|------|
| 読み取り | 不要 |
| 書き込み・新規作成 | 不要 |
| 削除 | 実況で報告してから実行 |

---

## Git操作

| 操作 | 承認 |
|------|------|
| コミット | 不要 |
| プッシュ | 実況で報告してから実行 |
| 強制プッシュ | 承認必要（破壊的変更） |

---

## API呼び出し

| 種類 | 承認 |
|------|------|
| GET系（Calendar閲覧、Slack履歴等） | 不要 |
| POST/PUT/DELETE（外部サービス） | 承認必要 |
| 実況チャンネルへのPOST | 不要 |

---

## Discord

| 操作 | 承認 |
|------|------|
| メッセージ読み取り | 不要 |
| 相談返信の送信 | 承認必要（外部送信） |
| ドラフト作成・内部分析 | 不要 |

---

## Google Calendar

| 操作 | 承認 |
|------|------|
| 閲覧 | 不要 |
| 予定作成・変更 | 承認必要 |

---

## Web検索・ファイル検索

承認不要。自由に使う。
